Prototype



Question

Which neighbourhood has maximum number of missing person incidents? and Is there a visible pattern between neighbourhoods reporting missing person incidents?

Answer

From this Prototype, it is clearly visible that highest number of missing person incidents are from "Tenderloin", "Mission" and "South of Market" neighbourhoods. The month reporting the highest number of incidents is a summer month(June - August) and the maximum number of incidents occur at 12:00 PM PST morning on Monday accross these neighbourhoods.

Encoding

Number of Incidents in each neighbourhood of SF is encoded over sequential color range. Darker the color, more number of incidents are in that particular neighbourhood.

Planned Interactivity

Details on demand interactivity, panning and zooming, and clicking over a particular neighbourhood in the map will show the hourly and monthly trend of that neighbourhood.(as shown in the prototypes above)

Peer Review Feedback

  • Color Encodings: The color scale is alright but there are big chunks of areas with the same color. The color between the different number of incidents can't easily distinguish.
  • Data Ink Ratio and Data Density: data density and data-ink ratio is also very good.
  • Planned Interactivity: You Can connect two prototypes using same filter.
  • Planned Interaction Effectiveness:I like it because it is informative
  • Gestalt Principles: It is very good.
  • Visual Encodings: The position and size of phototype is good.
  • Non Colour Encodings: The position, length and size, shape, color, and other visual attributes are good.
  • Design and Aesthetics: The website can be easily navigated. I pretty like the idea move the dashboard on the website.
  • Visualization Goal: which neighborhood has the maximum number of missing person incidents? I think to find the ratio will be much making sense.

Changes I Made

  • Changed the basemap of the choropleth map from "SF neighborhood" to "Police district" because of missing data for many neighborhoods within the report which results in lot of empty spaces within the map.
  • Changed the colour scheme so that colours can be easily figured out.
  • Encoded the choropleth map as percentage of missing person incidents accross police district instead of representing it as raw incident count.
  • Linked interactivities across prototypes as suggested in one of the comments.
