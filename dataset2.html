Historical Dataset

This is the older dataset. The dataset includes police incident reports filed by officers and by individuals through self-service online reporting for non-emergency cases. Reports included are those for incidents that occurred starting May 1, 2003 till May 15, 2018 and has been approved by a supervising officer. For up to date data starting in 2018, please go to the new dataset at: https://data.sfgov.org/d/wg3w-h783

The original dataset includes over 2.21M rows as of June 12, 2020. The original dataset includes 13 columns and each row is a incident record. The direct link to data set is: https://data.sfgov.org/Public-Safety/Police-Department-Incident-Reports-Historical-2003/tmnf-yvry. The full citation is :

Data acquired from https://datasf.org/opendata/ on June 12, 2020. Aggregated data on this page is made freely available under Open Data Commons Public Domain Dedication and License(https://opendatacommons.org/licenses/pddl/1.0/).
  1. IncidntNum: The number issued on the report, sometimes interchangeably referred to as the Case Number. The data type is plain text.
  2. Category: A category mapped on to the Incident Code used in statistics and reporting. Mappings provided by the Crime Analysis Unit of the Police Department. The data type is plain text.
  3. Descript: The description of the incident that corresponds with the Incident Code. These are generally self-explanatory. The data type is plain text.
  4. DayOfWeek: The day of week the incident occurred. The data type is plain text.
  5. Date: The date the incident occurred. The data type is date and time.
  6. Time: The time the incident occurred. The data type is plain text.
  7. PdDistrict: The Police District reflecting current boundaries (boundaries changed in 2015). Reference here: https://data.sfgov.org/d/wkhw-cjsf. Please note these are entered by officers and not based on the point. The data type is plain text.
  8. Resolution: The resolution of the incident at the time of the report. Can be one of: - Cite or Arrest Adult - Cite or Arrest Juvenile - Exceptional Adult - Exceptional Juvenile - Open or Active - Unfounded Note: once a report is filed the resolution does not change on the filed report later. Updates to a case will be issued later as Supplemental reports if there's a status change. The data type is plain text.
  9. Address: The address of the incident. The data type is plain text.
  10. X: The longitude coordinate in WGS84, spatial reference is EPSG:4326. Note, will be blank where geocoding was not possible. The data type is number.
  11. Y: The latitude coordinate in WGS84, spatial reference is EPSG:4326. Note, will be blank where geocoding was not possible. The data type is number.
  12. Location: The point geometry used for mapping features in the open data portal platform. Latitude and Longitude are provided separately as well as a convenience. Note, will be blank where geocoding was not possible. The data type is location.
  13. PdId: An identifier unique to the dataset. The data type is plain text.
